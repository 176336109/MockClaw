# 飞书电子表格API研究报告

## 研究概述
**研究时间**：2026年2月28日  
**研究目标**：深入研究飞书电子表格（多维表格）的API能力  
**研究方法**：权限分析 + API端点探索 + 现有文档研究

## 一、飞书电子表格API存在性确认

### 1.1 当前发现
基于研究，我发现了以下关键信息：

#### 1.1.1 飞书电子表格API确实存在
从飞书开放平台的URL结构可以看出：
- `https://open.feishu.cn/document/server-docs/docs/sheets-v3/overview` - 电子表格API文档
- 这表明飞书确实有专门的电子表格API（Sheets v3）

#### 1.1.2 与文档API的区别
- **文档API**：`docx/v1/` - 用于富文本文档
- **电子表格API**：`sheets/v3/` - 用于电子表格/多维表格
- **多维表格API**：`bitable/v1/` - 用于多维表格应用

### 1.2 API类型区分
1. **传统电子表格**：类似Excel的二维表格
2. **多维表格**：类似Airtable的数据库式表格
3. **文档内表格**：文档中的简单表格块

## 二、当前应用权限分析

### 2.1 已授权权限（81个）
从`feishu_app_scopes`工具返回的权限列表分析：

#### 2.1.1 文档相关权限（已拥有）
- `docx:document` - 文档API完整权限
- `docs:document.content:read` - 文档内容读取
- `docx:document:readonly` - 文档只读
- `docx:document:write_only` - 文档写入

#### 2.1.2 云盘相关权限（已拥有）
- `drive:drive.search:readonly` - 云盘搜索
- `drive:file:view_record:readonly` - 文件查看记录
- `space:document:move` - 文档移动

#### 2.1.3 缺失的电子表格权限
**重要发现**：当前应用**没有**以下权限：
- `sheet:*` - 电子表格相关权限
- `bitable:*` - 多维表格相关权限
- `spreadsheet:*` - 电子表格权限

### 2.2 权限获取方式
根据飞书开放平台的标准流程：

#### 2.2.1 申请新权限
1. **开发者后台**：访问飞书开放平台开发者后台
2. **应用管理**：找到目标应用
3. **权限管理**：添加所需权限
4. **提交审核**：提交权限申请
5. **企业管理员审批**：需要企业管理员批准

#### 2.2.2 所需权限范围
根据API类型，可能需要：
- **电子表格基础权限**：`sheet:spreadsheet:readonly` / `sheet:spreadsheet`
- **多维表格权限**：`bitable:app:readonly` / `bitable:app`
- **文件操作权限**：`drive:file:write` / `drive:file:readonly`

## 三、API功能特性研究

### 3.1 电子表格API能力（基于URL推断）

#### 3.1.1 核心操作
1. **创建电子表格**：创建新的电子表格文件
2. **读取数据**：读取单元格、范围数据
3. **写入数据**：更新单元格内容
4. **格式设置**：设置单元格格式、样式
5. **工作表管理**：添加、删除、重命名工作表

#### 3.1.2 高级功能
1. **公式支持**：支持Excel-like公式计算
2. **筛选排序**：数据筛选和排序功能
3. **图表创建**：生成图表和可视化
4. **数据验证**：单元格数据验证规则
5. **条件格式**：基于条件的格式设置

### 3.2 多维表格API能力

#### 3.2.1 核心概念
1. **应用(App)**：多维表格的容器
2. **表格(Table)**：数据表
3. **视图(View)**：数据的不同展示方式
4. **字段(Field)**：列定义
5. **记录(Record)**：行数据

#### 3.2.2 功能特性
1. **字段类型丰富**：文本、数字、日期、选项、人员、附件等
2. **视图多样化**：表格视图、看板视图、画廊视图等
3. **自动化流程**：自动化规则和触发器
4. **API集成**：Webhook和实时更新

## 四、适用场景分析

### 4.1 电子表格 vs 文档表格

#### 4.1.1 文档内表格（当前支持）
- **优点**：与文档内容集成，适合简单数据展示
- **缺点**：
  - 功能有限（不支持公式、筛选等）
  - 数据操作复杂
  - 不适合大规模数据处理
  - API支持有限（append接口不支持表格块）

#### 4.1.2 独立电子表格（需要新权限）
- **优点**：
  - 完整的数据处理功能
  - 支持公式计算
  - 数据筛选和排序
  - 图表和可视化
  - 更适合任务管理需求
- **缺点**：需要额外权限，与文档分离

### 4.2 任务管理适用性

#### 4.2.1 适合电子表格的场景
1. **项目任务跟踪**：甘特图、进度跟踪
2. **数据统计分析**：数据汇总、报表生成
3. **资源管理**：人员、时间、成本管理
4. **自动化报表**：定期数据更新和报告

#### 4.2.2 适合多维表格的场景
1. **CRM系统**：客户关系管理
2. **项目管理**：看板式任务管理
3. **内容管理**：文章、产品目录管理
4. **团队协作**：多人协同编辑和审批

## 五、技术实现分析

### 5.1 API端点结构推断

#### 5.1.1 电子表格API端点（推测）
```
# 电子表格管理
GET    /sheets/v3/spreadsheets                    # 获取电子表格列表
POST   /sheets/v3/spreadsheets                    # 创建电子表格
GET    /sheets/v3/spreadsheets/{spreadsheetToken} # 获取电子表格信息
PATCH  /sheets/v3/spreadsheets/{spreadsheetToken} # 更新电子表格

# 工作表操作
GET    /sheets/v3/spreadsheets/{spreadsheetToken}/sheets
POST   /sheets/v3/spreadsheets/{spreadsheetToken}/sheets
GET    /sheets/v3/spreadsheets/{spreadsheetToken}/sheets/{sheetId}

# 单元格操作
GET    /sheets/v3/spreadsheets/{spreadsheetToken}/values/{range}
PUT    /sheets/v3/spreadsheets/{spreadsheetToken}/values/{range}
POST   /sheets/v3/spreadsheets/{spreadsheetToken}/values:batchGet
POST   /sheets/v3/spreadsheets/{spreadsheetToken}/values:batchUpdate
```

#### 5.1.2 多维表格API端点（推测）
```
# 应用管理
GET    /bitable/v1/apps                          # 获取应用列表
POST   /bitable/v1/apps                          # 创建应用
GET    /bitable/v1/apps/{app_token}              # 获取应用信息

# 表格管理
GET    /bitable/v1/apps/{app_token}/tables       # 获取表格列表
POST   /bitable/v1/apps/{app_token}/tables       # 创建表格

# 记录操作
GET    /bitable/v1/apps/{app_token}/tables/{table_id}/records
POST   /bitable/v1/apps/{app_token}/tables/{table_id}/records
PATCH  /bitable/v1/apps/{app_token}/tables/{table_id}/records/{record_id}
DELETE /bitable/v1/apps/{app_token}/tables/{table_id}/records/{record_id}
```

### 5.2 认证方式
与文档API相同，使用：
1. **租户访问令牌**：企业级应用使用
2. **用户访问令牌**：用户级操作使用
3. **App Ticket**：自建应用需要

## 七、备选方案

### 7.1 如果电子表格API不可用

#### 7.1.1 使用文档内表格（当前方案）
- **优点**：无需额外权限，立即可用
- **缺点**：功能有限，不适合复杂数据处理
- **改进方向**：
  1. 优化表格块生成逻辑
  2. 开发文本模拟表格功能
  3. 提供表格模板库

#### 7.1.2 使用外部表格服务
1. **Google Sheets API**：功能强大，但需要Google账号
2. **Airtable API**：类似多维表格，但需要额外账号
3. **本地CSV/Excel**：本地文件处理，无需API

#### 7.1.3 数据库方案
1. **SQLite本地数据库**：轻量级，无需网络
2. **Supabase/PostgreSQL**：云数据库，功能完整
3. **自定义数据存储**：JSON/YAML文件存储

### 7.2 混合方案

#### 7.2.1 文档+外部数据
- **数据存储**：使用外部服务存储复杂数据
- **文档展示**：在飞书文档中展示摘要和链接
- **实时同步**：定期同步数据到文档

#### 7.2.2 渐进式增强
1. **阶段1**：使用文档内表格满足基础需求
2. **阶段2**：申请电子表格权限，开发基础功能
3. **阶段3**：实现高级功能和自动化

## 八、关键问题解答

### 8.1 飞书电子表格API真的存在吗？
**答案**：**存在**。飞书确实有专门的电子表格API（Sheets v3）和多维表格API（Bitable v1）。

### 8.2 我们当前有访问权限吗？
**答案**：**没有**。当前应用只有文档API权限，没有电子表格或多维表格相关权限。

### 8.3 如果没有，如何获取权限？
**答案**：需要通过飞书开放平台申请：
1. 登录飞书开放平台开发者后台
2. 找到目标应用
3. 在权限管理中添加所需权限
4. 提交申请并等待企业管理员审批

### 8.4 电子表格相比文档表格有什么实际优势？
**答案**：主要优势包括：
1. **功能完整**：支持公式、筛选、排序、图表等
2. **数据处理能力强**：适合大规模数据操作
3. **更适合任务管理**：看板视图、自动化流程
4. **API支持更好**：专门的API接口，功能更丰富
5. **协作性更强**：更适合团队协作和项目管理

### 9.2 风险评估

#### 9.2.1 技术风险
- **API限制**：可能存在调用频率、数据大小限制
- **功能差异**：飞书电子表格可能与Excel有功能差异
- **兼容性问题**：数据格式转换可能存在问题

#### 9.2.2 业务风险
- **权限审批**：企业管理员可能不批准权限申请
- **学习成本**：用户需要学习新的表格操作方式
- **迁移成本**：从现有方案迁移到新方案的成本

### 9.3 结论

飞书电子表格API是一个**存在且功能强大**的工具，但当前应用**缺乏访问权限**。相比文档内表格，电子表格在**数据处理能力、功能完整性和任务管理适用性**方面有明显优势。

**建议优先申请权限并开发基础功能**，这将显著提升OpenClaw在数据管理和任务处理方面的能力。即使权限申请受阻，也可以通过优化现有文档表格功能和开发混合方案来满足基础需求。
