# 问题解决流程检查清单

## 🎯 核心原则
**先查后做，复用优先，流程纪律**

## 🔄 标准解决流程

### 步骤1：查找以前的记忆
- [ ] 搜索记忆文件 (`memory_search`)
- [ ] 查看相关项目文件
- [ ] 检查脚本目录 (`scripts/`)
- [ ] 回顾最近的成功案例
- [ ] 确认是否有现成解决方案

### 步骤2：查看官方文档
- [ ] 查找相关工具文档
- [ ] 查看API文档
- [ ] 检查权限和配置
- [ ] 验证技术实现
- [ ] 确认最佳实践

### 步骤3：调试现有代码
- [ ] 运行现有解决方案
- [ ] 分析错误信息
- [ ] 检查日志和输出
- [ ] 验证环境配置
- [ ] 测试边界条件

### 步骤4：必要时创新
- [ ] 确认现有方案确实不行
- [ ] 在原有基础上扩展
- [ ] 保持向后兼容
- [ ] 记录修改原因
- [ ] 更新文档和记忆

## ⚠️ 常见错误（要避免）

### 错误1：跳过记忆查找
- ❌ 直接开始写新代码
- ✅ 先搜索记忆文件和现有脚本

### 错误2：重新发明轮子
- ❌ 完全重写已有功能
- ✅ 复用和扩展成功代码

### 错误3：忘记历史经验
- ❌ 不记得之前解决过类似问题
- ✅ 定期回顾记忆文件

### 错误4：流程纪律松懈
- ❌ 随意跳过步骤
- ✅ 严格遵守检查清单
