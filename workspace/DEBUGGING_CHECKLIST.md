# 问题解决检查清单

## 🎯 开始前准备

### [ ] 1. 心态调整
- 保持冷静，避免情绪化
- 接受不确定性，准备探索
- 相信问题可解，保持耐心

### [ ] 2. 问题定义
- 清晰描述现象（一句话）
- 确定影响范围和优先级
- 收集基本信息（环境、配置、版本）

## 🔍 阶段1：信息收集 (30分钟)

### [ ] 3. 日志收集
- 系统日志（最近24小时）
- 应用日志（错误级别以上）
- 访问日志（相关时间段）
- 监控告警（相关指标）

### [ ] 4. 环境检查
- 系统配置（配置文件）
- 依赖版本（软件版本）
- 资源使用（CPU、内存、磁盘、网络）
- 网络连接（端口、服务）

### [ ] 5. 历史查询
- 最近变更（代码、配置、部署）
- 类似问题历史记录
- 相关文档和知识库
- 团队经验分享

## 🧠 阶段2：分析诊断 (1-2小时)

### [ ] 6. 现象重现
- 最小重现步骤
- 环境隔离测试
- 控制变量实验
- 成功/失败条件明确

### [ ] 7. 时间线重建
- 事件发生时间点
- 相关系统变更时间
- 用户操作时间线
- 系统响应时间线

### [ ] 8. 假设生成
- 列出所有可能原因（头脑风暴）
- 按概率排序（高→低）
- 设计验证实验（每个假设）
- 预估验证时间（快速验证优先）

### [ ] 9. 认知偏差检查
- [ ] 确认偏差：是否只找支持证据？
- [ ] 锚定效应：是否被第一印象影响？
- [ ] 可得性启发：是否基于容易想到的例子？
- [ ] 群体思维：是否盲从他人观点？

## 🔧 阶段3：实验验证 (1-3小时)

### [ ] 10. 实验设计
- 一次只改变一个变量
- 明确成功/失败标准
- 准备回滚方案
- 记录实验过程

### [ ] 11. 快速验证
- 先验证高概率假设
- 使用最小可行测试
- 收集足够证据
- 及时调整方向

### [ ] 12. 根本原因确认
- 证据链完整（现象→原因）
- 排除其他可能原因
- 重现问题（确认原因）
- 修复验证（确认解决）

## 🚀 阶段4：解决方案 (30分钟-2小时)

### [ ] 13. 方案设计
- 短期修复（立即实施）
- 长期预防（系统改进）
- 风险评估（影响分析）
- 实施计划（步骤、时间、人员）

### [ ] 14. 方案实施
- 逐步实施（小步快跑）
- 实时监控（效果验证）
- 用户沟通（影响告知）
- 应急准备（回滚预案）

### [ ] 15. 效果验证
- 问题是否解决？（主要指标）
- 有无副作用？（其他指标）
- 用户反馈如何？（满意度）
- 系统稳定性？（监控数据）

## 📚 阶段5：学习改进 (30分钟)

### [ ] 16. 复盘总结
- 时间线回顾（发生了什么？）
- 决策评估（做了什么？为什么？）
- 效果分析（结果如何？）
- 教训总结（学到了什么？）

### [ ] 17. 知识沉淀
- 更新文档（解决方案）
- 改进流程（预防措施）
- 培训分享（团队学习）
- 工具优化（调试工具）

### [ ] 18. 系统改进
- 监控增强（提前发现）
- 告警优化（准确告警）
- 自动化（减少人工）
- 容错设计（优雅降级）

## 💡 特殊场景处理

### 紧急问题（P0级）
- [ ] 优先恢复服务（短期修复）
- [ ] 快速定位（关键路径）
- [ ] 简化流程（跳过非必要步骤）
- [ ] 事后详细分析（问题解决后）

### 复杂问题（多系统交互）
- [ ] 系统边界明确（责任划分）
- [ ] 接口检查（数据流验证）
- [ ] 协同调试（多团队协作）
- [ ] 端到端测试（全链路验证）

### 偶发问题（难以重现）
- [ ] 增加日志（详细记录）
- [ ] 监控增强（关键指标）
- [ ] 条件记录（环境信息）
- [ ] 用户反馈（使用场景）

## 🛠️ 工具使用指南

### 日志分析工具
- [ ] `grep` - 关键词搜索
- [ ] `tail -f` - 实时监控
- [ ] `jq` - JSON处理
- [ ] `awk` - 文本处理

### 监控分析工具
- [ ] 指标趋势图（时间序列）
- [ ] 异常检测（基线对比）
- [ ] 关联分析（多指标关联）
- [ ] 根因分析（影响链）

### 调试工具
- [ ] 断点调试（代码级别）
- [ ] 性能分析（CPU、内存）
- [ ] 网络分析（抓包、延迟）
- [ ] 存储分析（IO、容量）

## 📊 进度跟踪

### 时间分配建议
- 信息收集: 30分钟
- 分析诊断: 1-2小时  
- 实验验证: 1-3小时
- 解决方案: 30分钟-2小时
- 学习改进: 30分钟

### 里程碑检查
- [ ] 1小时: 是否找到明确方向？
- [ ] 2小时: 是否确认根本原因？
- [ ] 4小时: 是否实施解决方案？
- [ ] 8小时: 是否需要升级求助？

## 🚨 升级求助标准

### 需要求助的情况
- [ ] 超过4小时无进展
- [ ] 影响范围扩大
- [ ] 涉及多系统复杂交互
- [ ] 需要特殊权限或工具

### 求助准备
- [ ] 问题描述清晰
- [ ] 已尝试方法列表
- [ ] 当前进展和阻碍
- [ ] 需要帮助的具体内容
