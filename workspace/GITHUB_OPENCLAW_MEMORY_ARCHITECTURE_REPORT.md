## 🏆 核心发现：openclaw-memory-architecture

### 项目基本信息
- **仓库**: [coolmanns/openclaw-memory-architecture](https://github.com/coolmanns/openclaw-memory-architecture)
- **Stars**: 28 (快速增长中)
- **Forks**: 3
- **最后更新**: 2026-02-22
- **语言**: Python

### 架构设计：12层记忆系统

#### 基础层 (0ms延迟)
1. **会话上下文层** (~200K token窗口)
2. **工作记忆层** (`active-context.md`)
3. **长期记忆层** (`MEMORY.md` - 精选智慧)
4. **用户身份层** (`USER.md`)
5. **项目记忆层** (`memory/project-{slug}.md`)

#### 结构化层 (<1ms延迟)
6. **知识图谱层** (SQLite + FTS5)
   - 3,108个事实，1,009个关系，275个别名
   - 激活/衰减系统 (Hot/Warm/Cool层级)
   - 每日cron衰减调度

#### 语义搜索层 (7ms-4s延迟)
7. **语义搜索层**
   - 选项1: llama.cpp GPU (7ms, 多语言)
   - 选项2: QMD重排序 (~4s, 质量最佳)
   - 选项3: Ollama nomic-embed-text (61ms)
   - 支持100+种语言

#### 领域特定层 (~100ms延迟)
8. **领域RAG层**
   - 4,361个chunk，27个文档
   - 特定领域知识库
   - 每周重新索引

#### 插件层 (运行时)
9. **连续性插件** - 跨会话记忆
10. **稳定性插件** - 行为监控
11. **图记忆插件** - 实体注入
12. **其他插件层** - 可扩展

### 关键技术特性

#### 1. 多语言嵌入
- **模型**: nomic-embed-text-v2-moe (768维)
- **语言**: 100+种语言支持
- **延迟**: GPU上~7ms
- **部署**: llama.cpp Docker容器 + ROCm

#### 2. 知识图谱系统
- **规模**: 3,000+事实，1,000+关系
- **衰减**: Hot/Warm/Cool三层衰减
- **基准**: 100%召回率 (60/60查询)
- **存储**: SQLite + 全文搜索(FTS5)

#### 3. 插件生态系统
- **连续性插件**: 跨会话对话跟踪
- **稳定性插件**: 熵监控，原则对齐
- **图记忆插件**: 自动实体提取和注入

### 部署要求

#### 硬件推荐
| 组件 | 规格 |
|------|------|
| CPU | AMD Ryzen AI MAX+ 395 (16核/32线程) |
| RAM | 32GB DDR5 (与GPU统一) |
| GPU | AMD Radeon 8060S (96GB统一VRAM) |
| 存储 | 1.9TB NVMe |

#### 软件依赖
- Python 3.8+
- SQLite3
- Docker (用于llama.cpp)
- Node.js (用于插件)
