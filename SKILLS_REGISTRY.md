## Skill 生命周期

### 1️⃣ 发现阶段
- 发现候选Skill → 加入"候选Skill"表
- 填写：名称、来源、用途、优先级

### 2️⃣ 评估阶段
```markdown
## Skill评估清单（针对每个候选）
- [ ] 来源是否可信（官方repo/PyPI等）
- [ ] 代码是否有恶意内容（检查权限）
- [ ] 是否与现有Skill冲突
- [ ] 是否满足性能要求
- [ ] 文档是否完整
```

### 3️⃣ 安装阶段
```bash
# 安装前备份
git tag -a "before_SK_XXX" -m "Before installing Skill SK_XXX"

# 按照Skill的说明进行安装
# 在skills/installed/SK_XXX/ 文件夹中放入：
# - SKILL.md（使用说明）
# - config.json（配置）
# - version.txt（版本号）

# 更新本表，加入"已安装Skill"表
# 状态设为"✅ 活跃"
```

### 4️⃣ 维护阶段
- 每月第一周：检查所有已装Skill是否有安全更新
- 每周：检查Skill是否有过期或版本冲突
- 发现问题 → 立即更新"最后检查"字段 + 状态改为🟡

## 快速查询

### "我现在可以做什么？"
查看"已安装的Skill"表，寻找状态为"✅ 活跃"的行。

### "想要新能力？"
1. 在"候选Skill"表中添加一行
2. 创建对应的评估任务：`TASK_YYYYMMDD_XXX_评估_SK_名字`
3. 关联给合适的Agent

### "Skill出问题了？"
1. 记录在"已安装Skill"表中，状态改为🔴
2. 创建修复任务
3. 考虑是否应该临时下架
